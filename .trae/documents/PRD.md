## 1. 产品概述

企业内部LLM网关服务，统一管理多个大模型API接入，提供多模态Prompt模板管理与自动化质量评估能力。目标用户为企业AI应用开发者、数据科学家和运维管理人员。

- 解决企业多模型接入碎片化、Prompt管理混乱、输出质量难以量化的痛点
- 提供统一网关、可视化模板管理、自动化评估、全链路可观测的一体化解决方案

## 2. 核心功能

### 2.1 用户角色

| 角色 | 核心权限 |
|------|----------|
| 超级管理员 | 用户管理、模型配置、系统设置、全部数据访问 |
| 开发人员 | Prompt模板管理、评估管道配置、调用日志查看 |
| 运维人员 | 模型健康监控、日志检索、资源统计 |
| 普通用户 | 调用网关API、查看自己的调用记录 |

### 2.2 功能模块

1. **概览仪表盘**：调用量统计、Token消耗、模型分布、评估得分趋势
2. **模型管理**：模型供应商配置、API密钥管理、模型参数预设、健康状态监控
3. **Prompt模板管理**：模板CRUD、多模态嵌入（图片/表格/PDF）、版本管理、模板预览渲染
4. **评估管道**：断言规则配置、评估执行、评估结果详情、质量报告
5. **调用日志**：全量调用记录检索、详细调用信息、Token消耗与延迟分析
6. **API Playground**：在线调试接口、模板测试、实时评估

## 3. 核心流程

### 3.1 Prompt模板创建与调用流程

管理员创建多模态Prompt模板 → 配置变量与模态嵌入 → 选择目标模型预览渲染效果 → 配置评估断言规则 → 发布模板 → 业务系统通过网关API调用 → 网关根据模板渲染请求 → 转发至对应模型API → 接收响应后自动执行评估断言 → 记录调用日志与评估结果至Elasticsearch

```mermaid
flowchart LR
    A["创建多模态模板"] --> B["配置变量与模态"]
    B --> C["预览模型渲染格式"]
    C --> D["配置评估断言"]
    D --> E["发布模板"]
    E --> F["业务系统调用网关API"]
    F --> G["模板引擎渲染请求"]
    G --> H["转发至模型API"]
    H --> I["接收响应"]
    I --> J["执行评估断言"]
    J --> K["写入Elasticsearch"]
```

### 3.2 自动化评估流程

选择待评估模板 → 配置断言规则集 → 批量执行测试用例或在线调用 → 逐条运行断言（关键词/JSON/情感/自定义） → 生成评估得分与详细报告 → 记录历史版本对比

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深邃科技蓝 `#1e3a5f` 作为主色，搭配电光青 `#00d4ff` 作为强调色
- **辅助色**：成功绿 `#10b981`、警告橙 `#f59e0b`、危险红 `#ef4444`
- **背景主题**：深色主题，深灰底 `#0f172a` 配合细腻网格纹理背景
- **字体**：标题使用 `Space Grotesk`，正文使用 `JetBrains Mono` 等宽字体（契合代码/数据场景）
- **按钮风格**：微圆角、悬浮发光效果、渐变边框
- **布局风格**：侧边导航 + 顶部状态栏 + 主内容区，卡片式模块布局
- **视觉元素**：发光边框、渐变图表、微交互动画、数据可视化

### 4.2 页面设计概要

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|

- Desktop-first设计，主要面向1440px以上企业办公显示器

- 卡片布局自适应重排
